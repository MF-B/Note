- 实验
	- RAM字拓展和位拓展
	- MIPS RAM
		- 由4个(1Kx8)RAM组成
		- 数据总线为32位,但可根据Mode控制字实现字/半字/字节输入输出
		- 地址线为12位,高10位提供给4个RAM地址,低2位用于片选这4个RAM(字模式的时候均为1,半字模式时片选信号=0(1)时为低半字,=2(3)为高半字,比特模式时则等于片选信号)(硬件强制对齐)
	- MIPS 寄存器
		- 将Din和WE和W#用译码器＋三态门控制,0号寄存器恒0
		- 多路选择器控制输出RD1和RD2
- 理论
	- cache
		- 评价指标
			- 命中访问时间$t_c$ 
			- 缺失访问时间$t_m$
			- 命中率h=命中cache次数/(命中cache次数+从主存访问信息次数)
			- 访问效率=$t_c$/平均访问时间
		- 读流程
			- 根据地址查找表,找到则命中,数据送入cpu
			- 找不到则分配cache块,可能发生块替换。然后更新表
		- 写流程
			- 根据地址查表,命中则写数据到cache中,不命中则进入数据缺失状态
			- 新写入cache的数据是脏数据,需要进行写回(快,不一致)或写穿(慢,一致)
			- 数据缺失状态有两种处理方法,写分配法即分配cache块,再写数据到cache中,另一种则是直接将数据写入主存
		- 关键问题
			- 数据查找(如何判断数据是否在cache中)
				- 相联存储器(按内容访问)
			- 地址映射(主存数据块如何放入cache)
				- 全相联
					- 每一个主存块都可以映射到cache的任意行
					- 主存地址分为两段(块地址+块内偏移)
					- 需要并发比较器
				- 直接相联
					- cache行号i=主存块号j mod chche行数(等效为将主存按cache大小分区)
					- 主存地址分为3段(区地址+行索引+块内偏移)
					- 只需要一路比较器(结合行索引)
				- 组相联
					- 
			- 替换策略(cache满后怎么处理)
				- 先入先出算法
				- 最不经常使用算法
				- 近期最少使用算法
				- 随机替换算法
			- 写入策略(如何平衡速度与数据一致性)
				- 写回法
					- 可能导致DMA(直接内存访问)操作得到的数据不是最新数据
				- 写穿法
					- 无缓冲功能，降低cache功效
					- 多处理器时，每个cpu都有各自的cache,即使写穿也无法保证其他cpu的cache内容是最新的
		- 应用
			- 分离cache
				- 将数据cache和指令cache分离
			- 多级cache
				- L1cache通常在cpu内,容量小
				- L2cache在L1之后，容量增加
				- L3cache在L2之后，容量更大……
			- 软件cache
				- 只要存在分级(性能，成本，容量)差异，就可以用cache技术
					- 硬盘缓存
					- web cache
